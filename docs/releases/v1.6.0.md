Eidra Design System 1.6.0. All three packages (`@eidra/react`, `@eidra/icons`, `@eidra/tokens`) version together.

## Highlights

Two new components (Drawer, OTPField), a cascade-layer override model that makes DS components properly overridable from consumer code, a unified overlay-width policy that fixes full-width menus, and several "stop leaking into the host app" fixes (scoped box-sizing reset, opt-in Tailwind reset). Plus Base UI 1.6 and a batched dependency refresh.

## New components (`@eidra/react`)

- **Drawer**: an edge-anchored panel built on Base UI `Drawer` with swipe-to-dismiss. A `side` prop (`right` default, `left`, `top`, `bottom`) drives both the slide and the dismiss direction. Compound parts (Root / Trigger / Portal / Backdrop / Viewport / Popup / Title / Description / Close) plus Header / Body / Footer / CloseButton layout helpers. Token-styled and density-aware.
- **OTPField**: a one-time-code / PIN input rendering a row of single-character slots that auto-advance and accept pasted codes. Props: `length`, `validationType` (`numeric` / `alphanumeric` / `alpha`), `mask`, slot `size`. Integrates with `<Field>` for labels and validation.

## New features and API

- **Cascade-layer override model** (`@eidra/react`, ADR-0008). Component styles now ship wrapped in `@layer eidra`, so consumer `className` and Tailwind utilities override DS rules without inline `style` or `!important`. Import `@eidra/react/styles.css` before Tailwind's utilities import. On upgrade, an element previously held by a DS default may now pick up a consumer utility, so a quick visual check is worth it.
- **Unified overlay width policy** (`@eidra/react` + `@eidra/tokens`). Fixes `Select` / `Combobox` opening near-viewport-width menus. Width is decided in one place via a typed `width` prop: `anchor` (default for Select / Combobox / Autocomplete), `content` (default for Menu / ContextMenu / Menubar / FilterSelect), or `fill`. `--available-width` is now only ever a ceiling. New `OverlayWidth` type is exported. New tokens `--eidra-size-container-2xs` (11.25rem) and `--eidra-size-container-xs` (20rem) replace magic-number overlay widths.
- **ThemeProvider CSP nonce** (`@eidra/react`). New `nonce` and `cspDisableStyleElements` props; when set, the scope is wrapped in Base UI's `CSPProvider` so injected `<style>` / `<script>` tags carry a nonce (or are suppressed), enabling a strict `style-src` Content Security Policy. The default tree is unchanged.
- **Tailwind reset is now opt-in** (`@eidra/tokens` + `@eidra/react`, ADR-0009). The bridge (`tailwind.css`) now only maps Eidra tokens. The `@theme { --*: initial }` reset moved to a new export, `tailwind-reset.css`. Add it after `tailwindcss` and before `tailwind.css` only if you want Eidra-only utilities; omit it to keep Tailwind's defaults alongside the Eidra tokens.

## Fixes and improvements (`@eidra/react`)

- **Compact density now compacts horizontally**, not just vertically: `padding-inline` and `gap` step down on Button, Input, NumberField, Toggle, PeriodNavigator, Collapsible, FilterSelect, and DataGrid editable cells. RadioGroup gains a horizontal-orientation gap variant.
- **ScrollArea**: fixed the scrollbar thumb overflowing its track, and the thumb now deepens its colour while actively scrolling (Base UI 1.6 `data-scrolling` hook).
- **Box-sizing reset scoped to the Eidra subtree.** `styles.css` no longer forces global `box-sizing: border-box` on the host app; it applies to `.eidra-root` and portaled popups only. No API change. If you relied on it to reset your own markup, add your own reset.
- **Accordion** keyboard behavior follows Base UI 1.6 and the updated APG: headers are plain buttons in the Tab order, Enter/Space toggles, and arrow-key roving focus is removed.

## Internal and tooling (no published change)

- ESLint (flat config) + Prettier + Stylelint with a build-free CI `lint` gate, plus the genuine findings they surfaced.
- Storybook: Introduction landing page, pinned sidebar order, brand logo links to the hosted workshop.
- Stopped tracking the generated `packages/tokens/build/` scratch dir; narrowed the catalog-freshness CI guard.

## Upgrade notes at a glance

1. Import `styles.css` before Tailwind utilities (cascade layer).
2. The Tailwind reset is now opt-in (`tailwind-reset.css`).
3. The box-sizing reset no longer applies globally; add your own if you depended on it.
4. Delete inline `minWidth` / `width` overrides on overlay popups, and inline width/chrome overrides on DS components.
5. (Dev) vitest 4 config: `provider: playwright()` from `@vitest/browser-playwright`.
